## 1. 画作图片数据来源

### 1.1 推荐数据源（免费、合法、高清）

| 数据源 | 网址 | 特点 | 可用画作数量 |
|--------|------|------|------------|
| **Wikimedia Commons** | commons.wikimedia.org | 最大公共领域图片库，高清，免费商用 | 数十万幅 |
| **大都会艺术博物馆 Open Access** | metmuseum.org/art/collection | 超过47万件藏品开放下载，4K高清 | 47万+ |
| **芝加哥艺术博物馆** | api.artic.edu | 提供免费API接口，直接调用 | 5万+ |
| **国家美术馆 NGA** | nga.gov/open-access-images | 美国国家美术馆开放图片 | 5万+ |
| **欧洲数字图书馆 Europeana** | europeana.eu | 欧洲各大博物馆联合开放平台 | 百万级 |
| **故宫博物院** | digicol.dpm.org.cn | 部分藏品开放浏览（需关注授权政策） | 数万件 |

> ⚠️ **特别说明：** 卢浮宫官方图片受法国法律保护，建议使用 Wikimedia Commons 上对应的公共领域版本，而非直接从卢浮宫官网抓取。

### 1.2 各数据源接入方式

**大都会艺术博物馆（推荐首选）**
```
API地址：https://collectionapi.metmuseum.org/public/collection/v1/
使用方式：直接HTTP请求，无需注册，无需API Key
示例：获取画作详情
GET https://collectionapi.metmuseum.org/public/collection/v1/objects/436535
返回：标题、艺术家、年代、高清图片URL、版权信息等
```

**芝加哥艺术博物馆**
```
API地址：https://api.artic.edu/api/v1/artworks
使用方式：直接HTTP请求，无需注册
示例：搜索莫奈作品
GET https://api.artic.edu/api/v1/artworks?q=monet&fields=id,title,artist_display,image_id
```

**Wikimedia Commons**
```
API地址：https://commons.wikimedia.org/w/api.php
使用方式：HTTP请求，支持按分类批量获取
适合：补充其他来源没有的画作
```

## 2. 讲解文字内容来源

### 2.1 内容生产方式（三种并行）

#### 方式一：AI辅助生成（主力方式，效率最高）
- 使用 Python 脚本，批量调用 AI 大模型
- 输入：画作名称、艺术家、年代、风格
- 输出：300-500字故事化中文讲解
- 人工审核后入库
- **成本：** 每篇约￥0.1-0.3元，2000篇约￥200-600元

#### 方式二：参考权威资料改写
免费可参考的权威资料来源：

| 资料来源 | 网址 | 说明 |
|---------|------|------|
| 维基百科 | zh.wikipedia.org | 大量艺术家和画作词条，中文版 |
| 博物馆官方说明 | 各博物馆官网 | 权威，但需改写为故事化风格 |
| 谷歌艺术与文化 | artsandculture.google.com | 丰富的策展内容，参考改写 |

> ⚠️ **注意：** 参考以上来源时必须用自己的语言改写，不可直接复制，避免侵权。

#### 方式三：专业人士撰写
- 邀请艺术专业学生或自由撰稿人
- 适合重点画作（首页推荐、热门画作）
- 稿费建议：每篇￥50-150元

### 2.2 AI生成内容 Prompt 模板

```
你是一位专业的艺术讲解员，擅长用生动有趣的方式向普通大众介绍名画。

请介绍以下画作：
- 画作名称：{title}
- 艺术家：{artist}
- 创作年代：{year}
- 艺术风格：{style}
- 所在博物馆：{museum}

要求：
1. 字数300-500字
2. 用讲故事的方式，而非百科式描述
3. 包含：创作背景、画面解读、有趣故事或轶事、欣赏建议
4. 语言通俗易懂，适合没有艺术背景的普通读者
5. 适当加入能引发好奇心的细节
6. 输出纯文本，不使用markdown格式
```

## 3. 内容版权说明

### 3.1 什么样的画作可以免费使用？

**核心原则：** 作者逝世超过70年的作品进入公共领域，图片本身可免费使用。

| 情况 | 能否使用 | 说明 |
|------|---------|------|
| 作者逝世70年以上（如达芬奇、梵高、莫奈） | ✅ 可以 | 已进入公共领域 |
| 博物馆标注 CC0 或 Public Domain | ✅ 可以 | 明确开放使用 |
| 现代艺术家（逝世不足70年） | ❌ 不可以 | 需获得授权 |
| 博物馆官网图片（未标注授权） | ⚠️ 谨慎 | 建议用 Wikimedia 版本替代 |

## 4. 内容获取操作流程

### 4.1 新画作入库标准流程

```
第一步：确定目标画作清单（Excel表格）
    ↓
第二步：运行 fetch_artwork_images.py
        → 自动从大都会/芝加哥艺术博物馆API获取图片
        → 自动下载并压缩为WebP格式
        → 上传至云存储，获得图片URL
    ↓
第三步：运行 generate_description.py
        → 自动调用AI生成中文故事化讲解
        → 输出到待审核文件夹
    ↓
第四步：人工审核讲解内容
        → 检查文字是否通顺、有趣
        → 检查事实是否准确
        → 修改后标记"审核通过"
    ↓
第五步：运行 import_artworks.py
        → 将审核通过的内容批量写入云数据库
    ↓
第六步：在后台管理界面抽查验证
        → 确认图片显示正常
        → 确认语音生成正常
        → 上线发布
```

### 4.2 Python脚本说明

| 脚本文件 | 功能 | 运行频率 |
|---------|------|---------|
| `fetch_artwork_images.py` | 从开放API获取画作图片并上传云存储 | 每次新增画作时 |
| `generate_description.py` | 批量调用AI生成中文讲解文字 | 每次新增画作时 |
| `import_artworks.py` | 将审核通过的数据批量写入数据库 | 每次新增画作时 |
| `process_images.py` | 图片压缩、格式转换（转WebP） | 每次新增画作时 |
| `audit_content.py` | 生成待审核内容清单，方便人工审核 | 每次新增画作时 |
| `backup_database.py` | 数据库定期备份 | 每周自动运行 |

## 5. 内容管理后台设计

### 5.1 后台功能模块

作为开发者/运营者，您需要一个简单的后台管理界面来管理内容，建议使用**微信云开发自带的数据库控制台**结合一个轻量级后台页面实现：

| 功能模块 | 说明 |
|---------|------|
| **画作管理** | 新增/编辑/下架画作，查看浏览量 |
| **艺术家管理** | 新增/编辑艺术家档案 |
| **博物馆管理** | 新增/编辑博物馆信息 |
| **AI审核队列** | 查看用户触发的AI生成内容，一键通过/拒绝入库 |
| **AI接口配置** | 切换AI供应商，更新API Key，测试接口连通性 |
| **用户管理** | 查看用户数据，处理投诉 |
| **数据统计** | 浏览量、收藏量、搜索关键词排行 |

### 5.2 后台访问方式
- 初期使用微信云开发控制台（免开发，直接可用）
- V1.5阶段开发专属后台管理页面

## 7. 内容质量控制标准

### 画作内容审核清单

每篇讲解内容入库前需满足：

- [ ] 文字300-500字
- [ ] 故事化叙述，非百科式罗列
- [ ] 无错别字、语病
- [ ] 事实准确（艺术家、年代、博物馆无误）
- [ ] 语言通俗，无过多专业术语
- [ ] 图片清晰，宽度≥1200px
- [ ] 图片版权确认为公共领域或CC0
- [ ] 艺术家姓名、画作标题中英文均已填写
