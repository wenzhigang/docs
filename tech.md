## 1. 技术架构总览

### 1.1 整体架构图

```
┌─────────────────────────────────────────┐
│            用户端（微信小程序）            │
│  Taro框架 + React + TypeScript           │
└────────────────┬────────────────────────┘
                 │ HTTPS请求
┌────────────────▼────────────────────────┐
│         微信云开发（CloudBase）           │
│  ┌──────────┐ ┌──────────┐ ┌─────────┐ │
│  │  云函数   │ │  云数据库 │ │  云存储  │ │
│  │ Node.js  │ │  NoSQL   │ │图片/音频 │ │
│  └──────────┘ └──────────┘ └─────────┘ │
└────────────────┬────────────────────────┘
                 │
┌────────────────▼────────────────────────┐
│            第三方服务                    │
│  腾讯云TTS（语音）  腾讯云图像搜索（识图） │
└─────────────────────────────────────────┘
```

### 1.2 技术选型说明

| 层级 | 技术方案 | 选择理由 |
|------|---------|---------|
| 前端框架 | Taro 3.x + React | 一套代码可编译微信小程序，未来可扩展至App |
| 开发语言 | TypeScript | 类型安全，减少运行时错误，便于团队协作 |
| 后端服务 | 微信云开发 CloudBase | 无需自建服务器，与微信生态无缝集成，运维成本低 |
| 数据库 | 云数据库（MongoDB风格） | 灵活的文档结构，适合内容型产品 |
| 文件存储 | 云存储 | 高清图片、音频文件存储，自带CDN加速 |
| AI语音 | 腾讯云TTS | 与微信生态同源，接入简单，中文音质优秀 |
| 图像识别 | 腾讯云图像搜索 | 支持自定义图库，适合画作以图搜图场景 |
| 状态管理 | Zustand | 轻量级，适合小程序体量 |
| UI组件库 | NutUI（京东出品） | 专为Taro优化，组件丰富，中文文档完善 |

## 2. 前端设计（微信小程序）

### 2.1 目录结构

```
/src
├── pages/                  # 页面
│   ├── home/               # 首页
│   ├── explore/            # 探索页
│   │   ├── museum/         # 博物馆列表/详情
│   │   └── artist/         # 艺术家列表/详情
│   ├── artwork/            # 画作详情页
│   ├── scan/               # 拍照识图页
│   ├── education/          # 教育版页面
│   └── profile/            # 个人中心
├── components/             # 公共组件
│   ├── ArtworkCard/        # 画作卡片组件
│   ├── ArtistCard/         # 艺术家卡片组件
```

### 2.2 页面清单

| 页面 | 路径 | 说明 |
|------|------|------|
| 首页 | `/pages/home/index` | 推荐内容、今日画作、热门艺术家 |
| 探索页 | `/pages/explore/index` | 博物馆/艺术家分类入口 |
| 博物馆列表 | `/pages/explore/museum/list` | 所有博物馆列表 |
| 博物馆详情 | `/pages/explore/museum/detail` | 博物馆介绍+馆藏列表 |
| 艺术家列表 | `/pages/explore/artist/list` | 所有艺术家列表 |
| 艺术家详情 | `/pages/explore/artist/detail` | 艺术家故事+作品画廊 |
| 画作详情 | `/pages/artwork/detail` | 高清图+讲解+语音+标注 |
| 拍照识图 | `/pages/scan/index` | 调用摄像头，识别画作 |
| 教育版首页 | `/pages/education/index` | 课程包列表+测验入口 |
| 个人中心 | `/pages/profile/index` | 收藏夹+学习记录+徽章 |

### 2.3 底部导航栏

```
首页 | 探索 | 拍照识图（中间突出按钮）| 教育 | 我的
```

### 2.4 核心组件说明

**ArtworkCard（画作卡片）**
- 展示画作缩略图、名称、艺术家、博物馆
- 支持懒加载，避免页面卡顿

**AudioPlayer（语音播放器）**
- 悬浮于画作详情页底部
- 支持播放/暂停/进度拖拽
- 自动缓存已播放音频

**ImageViewer（高清图片查看器）**
- 支持双指缩放、单指平移
- 支持点击局部区域触发细节标注弹窗

## 5. API接口设计

### 5.1 接口规范

- 所有接口统一返回格式：
```json
{
  "code": 0,
  "message": "success",
  "data": {}
}
```
- 错误码：`0` 成功，`1001` 参数错误，`1002` 未登录，`5000` 服务器错误

### 5.2 核心接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/artwork/list` | GET | 获取画作列表，支持 museum_id / artist_id / page 参数 |
| `/artwork/detail/:id` | GET | 获取画作详情 |
| `/artist/list` | GET | 获取艺术家列表 |
| `/artist/detail/:id` | GET | 获取艺术家详情 |
| `/museum/list` | GET | 获取博物馆列表 |
| `/museum/detail/:id` | GET | 获取博物馆详情 |
| `/scan/identify` | POST | 上传图片进行画作识别 |
| `/audio/generate` | POST | 生成指定画作的语音朗读 |
| `/user/login` | POST | 微信登录 |
| `/user/favorite` | POST | 添加/取消收藏 |
| `/user/profile` | GET | 获取用户信息 |

## 6. 第三方服务集成

### 6.1 腾讯云TTS（语音合成）

- **用途：** 将画作文字讲解转换为语音音频
- **接入方式：** 云函数中调用腾讯云SDK
- **音色建议：** 使用"智瑜"（温柔女声）或"智云"（成熟男声），可由用户切换
- **生成策略：** 首次请求时生成并缓存至云存储，后续直接读取缓存
- **预估成本：** 每100万字符约￥300，MVP阶段成本极低

### 6.2 腾讯云图像搜索（拍照识图）

- **用途：** 用户拍照后识别对应画作
- **接入方式：** 预先将所有收录画作图片导入腾讯云图库，用户拍照后以图搜图
- **识别流程：**
```
用户拍照
  ↓
压缩图片（小于1MB）
  ↓
上传至云函数
  ↓
调用腾讯云图像搜索API
  ↓
返回相似度最高的画作ID
  ↓
跳转至对应画作详情页
```
- **识别准确率：** 光线充足、正面拍摄时准确率可达90%以上
- **失败处理：** 识别失败时提示用户"未能识别，请尝试正面拍摄"，并提供手动搜索入口

### 6.3 微信登录

- 使用微信小程序原生登录能力（`wx.login`）
- 获取用户 openid 作为唯一标识
- 不强制要求用户授权昵称/头像，保护用户隐私

## 7. 安全设计

| 安全项 | 措施 |
|--------|------|
| 用户身份验证 | 微信openid + 云开发鉴权，所有接口验证登录态 |
| 数据权限 | 云数据库设置权限规则，用户只能读写自己的数据 |
| 图片版权 | 仅使用公共领域画作（作者逝世70年以上），来源标注清晰 |
| 防刷接口 | 云函数设置调用频率限制，防止恶意请求 |
| 内容安全 | 用户生成内容（如笔记）接入微信内容安全检测API |

## 8. 性能优化策略

| 优化项 | 方案 |
|--------|------|
| 图片加载 | 懒加载 + WebP格式 + 云存储CDN加速 |
| 首屏速度 | 分包加载，首页体积控制在500KB以内 |
| 音频缓存 | 已播放音频缓存至本地，避免重复请求 |
| 数据缓存 | 列表页数据本地缓存5分钟，减少重复请求 |
| 分页加载 | 所有列表页采用下拉分页，每页加载20条 |

## 10. 版本发布流程

```
开发分支（dev）
    ↓ 功能开发完成
代码审查（自查）
    ↓ 确认无误
合并至主分支（main）
    ↓
提交微信小程序审核
    ↓ 审核通过（通常1-3个工作日）
正式发布上线
```

### 分支命名规范
```
main          ← 正式版本
dev           ← 开发版本
feature/xxx   ← 新功能分支（如 feature/scan-artwork）
fix/xxx       ← 修复分支（如 fix/audio-player）
```
